= Keep your AI Data Engine system up to date
:icons: font
:imagesdir: ../media/

[.lead]
Keep AI Data Engine (AIDE) system components up to date to maintain performance and access to new features. Update components after deployment, when new software or firmware becomes available, when you add or replace nodes, or periodically for feature updates.

== AIDE system components
There are two main components that require updates in an AI Data Engine system: ONTAP software and AI Data Engine software, which includes DCN firmware.

=== ONTAP Software
ONTAP is the operating system that runs on NetApp storage systems, including those used in AI Data Engine deployments. Keep ONTAP current to preserve system stability, security, and compatibility with AIDE components. AIDE components are updated separately.

=== AI Data Engine software
DCN firmware and AIDE Console software updates are distributed together as a single package (.tgz) and are not embedded in ONTAP images.

.DCN firmware updates
Update firmware to ensure the proper functioning of hardware components within the AI Data Engine system, including data compute nodes (DCNs), storage controllers, network interfaces, and other critical devices.

.AI Data Engine software
Update the AI Data Engine software regularly to obtain new features, performance improvements, and bug fixes. AIDE software includes the AI Data Engine Console, vectorization, and guardrail components.

== Compatibility matrix
Ensure compatibility between ONTAP, DCN firmware, and AI Data Engine software when planning updates.

AI Data Engine software is released as "U releases". An AIDE major release is a "U0" release and subsequent updates will be "U1" and later. AIDE major releases and update releases within that major release are compatible with the corresponding ONTAP major releases and patch releases. For example, AIDE 9.18.1 U0 (major release) and AIDE 9.18.1 U1 (update) are compatible with ONTAP 9.18.1 GA (major release) and all ONTAP 9.18.1 P (patch) releases.

.Upgrade paths

* *AIDE major releases*: Upgrade sequentially between major versions (for example, 9.18.1 to 9.19.1). You cannot skip major versions.
* *AIDE update releases*: You can skip U releases within the same major version (for example, 9.18.1 U0 to 9.18.1 U2). When upgrading to a new AIDE major version, upgrade any AIDE U version to any U version in the corresponding next major version (for example, AIDE 9.18.1 U1 to 9.19.1 U2).

[cols="1,1",options="header"]
|===
| AIDE Releases | Supported ONTAP Version

| 9.18.1 and all 9.18.1 Ux releases | 9.18.1 GA
| 9.18.1 and all 9.18.1 Ux releases | 9.18.1 Px
|===

NOTE: "Px" denotes ONTAP patch releases (for example, 9.18.1 P1). "Ux" denotes AIDE update releases (for example, 9.18.1 U1).

.Release types and scope

* ONTAP major releases and AIDE major releases introduce new features, APIs, or changes that affect ONTAP integration.
* ONTAP patch releases and AIDE update releases contain fixes and updates that do not impact ONTAP integration.

== Revert limitations
AIDE systems do not support revert operations for DCN firmware, AIDE software updates, or ONTAP on AFX storage systems. After you install an update or upgrade, you cannot revert to a previous version. Review the release notes and compatibility requirements for each component before you update or upgrade.

== Understand the update process
AI Data Engine DCN and AIDE Console software updates can be managed through ONTAP System Manager, which provides options for partially automatic or manual updates.

AIDE does not support fully automatic software updates. If you use ONTAP automatic updates, the system can notify admins about available updates. Admins must then complete the process manually.
