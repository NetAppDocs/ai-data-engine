= Observe data collections in AI Data Engine
:icons: font
:imagesdir: ../media/

[.lead]
After data engineers create and publish data collections from workspaces, both storage administrators and data engineers need visibility into their status, size, and impact on the AI Data Engine cluster.

If you're a storage administrator or data engineer, you can observe data collections across System Manager and AIDE Console.

== Before you begin

* You need either _storage administrator_ privileges in System Manager or _data engineer_ privileges in AI Data Engine Console to observe data collections.
* At least one workspace exists with successfully extracted metadata.
* Data engineers have created and published at least one data collection from AI Data Engine Console.
* The `AIPVS - Vectorization and Guardrails` license is installed and inferencing features are enabled, so that vectorization and retrieval endpoints are active.

== View all data collections (admin view in System Manager)

System Manager provides a cluster-wide view of data collections and their footprint but does not allow admins to create or modify them.

.Steps
. In System Manager, navigate to *Data Engine > Data collections*.
. Review the inventory summary at the top of the page:
.. Total number of data collections by status (for example, `Ready`, `Publishing`, `Error`).
.. Total space consumed by the vector database across all collections.
.. Vector space as a percentage of overall cluster capacity.
. In the data collection grid, review:
.. Collection name and description.
.. Associated workspace.
.. Status (for example, `Publishing`, `Ready`, `Failed`).
.. Collection size.
.. Last updated time.

.Result
You now have a high-level view of all data collections in the cluster and their storage impact. Use this view to identify collections that are large, stale, or stuck in a non-ready state.

TIP: Use the search and filter controls to focus on a single workspace, a status (such as `Error`), or a specific collection name.

== Inspect a single data collection (admin view)

For deeper insight into a particular data collection, open its details page in System Manager.

.Steps
. From *Data Engine > Data collections*, select the name of a data collection.
. In the *Overview* tab, review:
.. Workspace association.
.. Definition summary (for example, filtered file types or metadata criteria).
.. Current status and last publish time.
.. Reported size and item counts.
. If the collection is in `Publishing` or `Processing` state, monitor the progress indicator on this page.
. If an error is reported, note the error message and cross‑check with the *Activity* page for related jobs and events.

.Result
You can see whether the collection is actively being updated, when it last completed, and whether any failures are blocking RAG usage.

== Monitor collection-related jobs and events

Jobs that build and update collections can be monitored from the cluster-wide *Activity* page and from the workspace details.

.Steps
. In System Manager, go to *Data Engine > Activity*.
. On the *Events* tab:
.. Filter by type (for example, workspace, data collection) or severity.
.. Expand any event related to data collections (for example, "Data collection publish failed") to see more details.
. On the *Jobs* tab:
.. Filter by *Workspace* or *Job type* to focus on data collection indexing and publishing jobs.
.. For each job, open the peek view to see:
*** Progress percentage.
*** Start and end times.
*** Any reported error messages or warnings.
. Optionally, navigate back to the affected workspace and open its *Activity* tab to see events and jobs scoped only to that workspace.

.Result
You can track the lifecycle of data collections, identify stalled or failed jobs, and gather contextual information to pass to data engineers or support.

TIP: When a data collection remains in `Publishing` state for an extended period, check for a corresponding long-running job in the Activity page before assuming a failure.

== Observe data collections from AIDE Console (data engineer view)

Data engineers typically monitor data collections directly from AIDE Console, where they are created and published.

.Steps
. Log in to AIDE Console as a data engineer.
. Go to *Workspaces* and select the desired workspace.
. Open the *Data collections* tab.
. For each collection:
.. Check the status (`Draft`, `Publishing`, `Ready`, `Failed`).
.. Review definition details (filters, included file types, classifier options, embedding settings).
.. Inspect timestamps for last publish or update.
. If needed, open job details or logs (where available) to understand failures or incomplete runs.

.Result
Data engineers can iterate on collection definitions and re‑publish them while monitoring status and health, without involving storage administrators.

== Understand vector footprint and capacity impact

Large data collections can grow the vector database significantly. Use the System Manager inventory view to keep vector space under control.

.Steps
. In System Manager, open *Data Engine > Data collections*.
. Review:
.. Total vector database size across all collections.
.. Vector space as percentage of total cluster capacity.
. Sort the collection grid by *Size* to identify the largest contributors.
. Coordinate with data engineers to:
.. Retire unused collections.
.. Adjust filters to reduce scope where appropriate.
.. Schedule re‑publishes during lower‑usage periods to minimize performance impact.

.Result
You can proactively manage capacity consumption and prevent unbounded growth of embeddings from rarely used or overly broad data collections.
