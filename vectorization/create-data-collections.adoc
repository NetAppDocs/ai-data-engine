= Create data collections in AI Data Engine Console
:icons: font
:imagesdir: ../media/

[.lead]
//Workstream 2 focused
Data collections are the core RAG building blocks in AI Data Engine (AIDE). As a data engineer, you define which files and objects belong in a collection, configure embedding and indexing options, and publish the collection so that applications can query it through a retrieval endpoint.

== Before you begin

* You need _data engineer_ privileges in AI Data Engine Console and access to at least one workspace.
* You have access to at least one workspace with metadata extracted and in `Ready` state.
* You have explored the workspace metadata and identified queries or filters that define meaningful subsets of data.
* The `AIPVS - Vectorization and Guardrails` license is installed and inferencing features are enabled.

== Create a data collection from workspace metadata

.Steps
. In AIDE Console, navigate to *Workspaces* and select the workspace that contains your target data.
. Go to the *Data collections* tab.
. Select *Create data collection*.
. In the *Define collection* step:
.. Enter a *Name* and *Description* for the collection (for example, `Support_KB_RAG_EN`).
.. Choose whether the collection should be:
*** *Static*: A fixed snapshot of the matched items.
*** *Dynamic* (if supported): Automatically tracks new items that match the same criteria during workspace refreshes.
. Specify the source subset:
.. Start from a saved metadata query, or
.. Build a query in this step by applying filters (path, file type, timestamps, and other attributes).
. Review the estimated item count and approximate size.

.Result
You have defined the logical scope of the data collection. The actual embeddings and vector index will be created during publishing.

TIP: Create small, focused collections (for example, per use case or domain) rather than a single "everything" collection. This improves retrieval relevance and manageability.

== Configure embedding and indexing settings

.Steps
. In the *Embedding and indexing* step of the collection wizard:
.. Choose an *Embedding model* from the supported list.
.. Select embedding dimensionality if the model offers multiple options (for example, 512, 768, 1024).
.. Configure *Chunking*:
*** Choose granularity (sentence, paragraph, page, or full file).
*** Adjust max tokens or characters per chunk, if available.
.. (Optional) Set language or MIME-type filters if you want to exclude certain file types from embedding.
. Review any system-provided estimates:
.. Expected embedding size.
.. Expected vector DB footprint.

.Result
The collection now has a complete indexing configuration that AIDE will use to generate embeddings and build the vector index.

NOTE: Changing embedding settings after publishing typically requires a re-publish of the collection, which re-processes the content.

== Publish a data collection

Publishing runs the indexing pipeline: canonicalization, classification (if enabled), embedding generation, and vector DB indexing.

.Steps
. From the *Data collections* tab for the workspace, select your new collection.
. Click *Publish* (or *Start indexing*).
. Confirm the settings and start the job.

.Result
The collection enters a `Publishing` or `Processing` state. AIDE Console and ONTAP System Manager both show job progress:

* In AIDE Console, the collection's status updates and a progress bar appears.
* In ONTAP System Manager, corresponding jobs appear under *Data Engine > Activity > Jobs* and under the workspace's *Activity* tab.

TIP: For large collections, consider scheduling initial publish and major re-publishes during off-peak times to minimize resource contention.

== Monitor publish progress and resolve failures

.Steps
. In AIDE Console, open the collection detail view.
. Observe:
.. Current status (`Publishing`, `Ready`, `Failed`).
.. Progress indicators and counts.
. If the collection fails:
.. Review the error message in the collection detail.
.. Open the job details or logs (where available) to identify causes such as:
*** Unsupported file types.
*** Access or permission errors.
*** Resource limits or timeouts.
.. Adjust the collection definition or workspace configuration as needed (for example, exclude problematic paths or file types).
.. Publish the collection again.

.Result
The collection reaches the `Ready` state and is available for use by downstream applications and data scientists.

== Update or delete a data collection

Over time you might need to refine or retire collections.

=== Update a collection definition

.Steps
. In AIDE Console, open the workspace and select the *Data collections* tab.
. Select the collection you want to modify.
. Choose *Edit*.
. Adjust:
.. Filters (paths, file types, classification tags).
.. Static/dynamic behavior (if supported).
.. Embedding and chunking settings.
. Save your changes.
. Publish the collection again so that the new definition and embeddings take effect.

.Result
A new indexing job runs with the updated configuration, and the collection returns to a `Ready` state when complete.

=== Delete a collection

.Steps
. From the *Data collections* tab, select the collection.
. Choose *Delete*.
. Confirm the deletion.

.Result
The collection definition and its embeddings are removed from AI Data Engine. Applications attempting to query the former retrieval endpoint will fail once the collection is removed.

CAUTION: Deleting a collection is permanent. Ensure that no production application still depends on the collection's retrieval endpoint before deleting it.
