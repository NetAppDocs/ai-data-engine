= AIDE data engineers and component interactions
:icons: font
:imagesdir: ../media/

[.lead]
As a data engineer, you use the AI Data Engine Console to explore workspaces, create data collections, manage embedding pipelines, and integrate AI/ML workflows. Your role focuses on transforming raw data into AI-ready datasets.

== Data engineer component access

[cols="2h,3", options="header"]
|===
|Component
|Data engineer access

|ONTAP System Manager
|No access

|AI Data Engine Console
|Manage (create, edit, delete)

|ONTAP REST API
|Manage (create, edit, delete)

|Workspaces
|View/Use (read-only, consume for workflows)

|Data collections
|Manage (create, edit, delete)

|Guardrails
|View/Enforce (read-only, operate within)

|Metadata catalog
|Query/Use (consume for workflows)

|Vector database
|Manage embeddings/Search (create, configure, query)

|Classifiers
|Apply/Review (use existing, view results)

|===

== AIDE system components

=== Workspaces

You receive access to assigned workspaces. You use the AI Data Engine Console to explore, query, and understand available data sources. You build data collections from workspace data and manage data pipelines.

=== Metadata catalog

You use the metadata catalog in the AI Data Engine Console to discover, filter, and understand data sources before building data collections. You run queries to identify relevant files and objects.

=== Classifiers

You apply classifiers to data collections, review classification results, and use them to filter or annotate data for AI workflows.

=== Data collections

You create and manage data collections by querying the workspace's metadata catalog in the AI Data Engine Console. You filter data by tags, classification, or other attributes. You manage lifecycle, retention, and governance policies for collections. You publish collections as RAG endpoints for AI/ML use.

=== Vector database

You trigger embedding pipelines for data collections, select models and parameters, and monitor vectorization status in the AI Data Engine Console. You use APIs to perform semantic search and integrate retrieval endpoints into AI applications.

=== Guardrails

You work within the guardrails attached to workspaces and data collections. You may request changes to guardrail policies for workflow needs but cannot override them. Guardrails ensure compliant and secure data processing.

=== Retrieval endpoint (RAG endpoint)

You publish data collections as RAG endpoints, configuring chunking and embedding parameters. You manage RBAC access and provide endpoint details for integration with AI applications.
