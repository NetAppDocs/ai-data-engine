= Quick start for AI Data Engine
:icons: font
:imagesdir: ../media/

[.lead]
To get up and running with your AI Data Engine system, you need to install your hardware components, set up your cluster, set up data access from your hosts to the storage system, and provision your storage.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png[One] Will AIDE be installed with a new or existing AFX cluster?

[role="quick-margin-para"]
You need to decide if AIDE and AFX will be installed together at the same time or if AIDE will be integrated with an existing AFX cluster.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png[Two] Install and set up your hardware

[role="quick-margin-para"]
link:../install-setup/install-setup-workflow.html[Install and set up] your AFX hardware, including DCN compute nodes.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png[Three] Set up your cluster

[role="quick-margin-para"]
Use System Manager to guide you through a quick and easy process to set up your cluster. Select either link:../install-setup/cluster-setup-afx.html[Set up AIDE with an AFX cluster].

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png[Four] Set up workspaces and data access

[role="quick-margin-para"]
link:../workspaces/create-data-container.html[Set up a workspace and users for that workspace who can access AI Data Engine data].

.What's next?
You can now use System Manager to manage your AI Data Engine and get your data engineers and data scientists started with their workspaces and configurations.
