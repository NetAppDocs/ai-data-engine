= Quick start for AI Data Engine
:icons: font
:imagesdir: ../media/

[.lead]
To get up and running with your AI Data Engine system, you install your hardware components, set up your cluster, set up data access from your hosts to the storage system, and provision your storage.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png[One] Install and set up your hardware

[role="quick-margin-para"]
link:../install-setup/install-setup-workflow.html[Install and set up] your AFX hardware, including DCN compute nodes.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png[Two] Set up your cluster

[role="quick-margin-para"]
Use System Manager to guide you through a quick and easy process to link:../install-setup/cluster-setup.html[set up your AI Data Engine cluster].

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png[Three] Set up data for workspaces

[role="quick-margin-para"]
Use System Manager to link:../workspaces/create-data-container.html[set up a data container] for data you intend to use with AI Data Engine.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png[Four] Set up workspaces and data access

[role="quick-margin-para"]
link:../workspaces/create-data-container.html[Set up a workspace and users for that workspace who can access AI Data Engine data].

.What's next?
You can now use System Manager to manage your AI Data Engine and get data engineers and data scientists started with their workspaces and configurations.
